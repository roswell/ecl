
* Trick 1: lazily compile the discriminator

When COMPUTE-DISCRIMINATING-FUNCTION is called, we install an invalidated
generic function (instead of compiling it immediately), that will compile the
function on first call.

Thanks to this we avoid repeated unnecessary recompilations when we add new
methods or when the function is never actually called. This greatly boosts
performance of benchmarks CLOS/defclass and CLOS/defmethod.

This, in effects, makes loading much faster (even compared to baseline) and the
system feels much more responsive.
